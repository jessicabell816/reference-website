Looking Back at CSS

Linking CSS to HTML

  • href: Used to point to the location of the CSS file (the path).
  • rel: This attribute is used to define this item as a stylesheet.
  • Example: rel="stylesheet" href="../css/style.css

Border Box

  • Resets the layout math.
  • Put at the top of EVERY CSS file!
  • html{ box-sizing: border-box; } *,*::before, *::after { box-sizing:inherit; }

Syntax of CSS

  • Rule set: Composed of a selector and one or many declarations surrounded by curly braces {}.
  • Selector (ex.h1): Identifies the HTML element being targeted for styling.
  • Declarations: Assigns a value to a property; declarations must end with a semi-colon
  • Property: The type of design you want to add (ex. width, color, border, etc.).
  • Value: Accepted value for property (ex. 30px, blue, etc.).

Properties, Decorations & States

Font Properties

  • font-family: Used to style the typeface used in the website.
  • font-size: For adjusting the size of the font: a single number with a unit (rem [the best choice], em, px).
  • font-weight: Sets how thick or thin characters in text should be displayed (ex: bold, normal, 700 (bold))
  • font-style: Specifies font style for text (ex: italic or normal).
  • line-height: For adjusting the line-height: a single number, a multiplier that multiplies the font-size (ex: 1.5)

Text Decorations

  • text-transform: Change capitalization such as uppercase, lowercase, capitalize.
  • text-decoration:Specifies the decoration added to text (ex: none, underline, line-through, overline).
  • text-align: Specifies the horizontal alignment of text in an element (ex: left, center, right, justify).
  • text-indent: Indent the first line of text like traditionally printed materials (ex: 1rem, 2em, 20px, 25%).
  • text-shadow: Adding a shadow to the text. Specified as: horizontal-offset vertical-offset blur-radius colour (ex.text-shadow: 2px 2px #ff0000;).
  • list-style-type: Removes default decorations from ordered and unordered lists.

Link States

  • Default: (a:link)
  • Visited: For styling the link after it’s been visited by the user (a:visited).
  • Hover: For styling the link when the mouse is sitting over it (a:hover).
  • Focus: For styling the link when the keyboard has tab-focused it (a:focus).
  • Active: For styling the link when the mouse button is pressed down on it (a:active).

Targeting HTML

Selectors

  • Tag: Select all the tags of the same type (ex. html {}).
  • Class (.): Select an element that has a class; needs a matching class in the HTML (ex. .nav {}).
  • ID (#): Select an element that has the ID; needs a matching class in the HTML (ex. #heading {}).
  • Descendant (space): Select an element that's a descendant of another element (ex. ul li {}).
  • Child (>): Select an element directly inside another element (ex. ul > li).
  • Adjacent Sibling (+): Select an element immediately beside another element (ex. h1 + p).
  • General Sibling (~): Select an element that's at the same level (ex. h1~p).
  • Atrribute ([]): Select an element by it's attribute. Good for styling links differently if they're external (ex. [href^="http"]).

The CSS Box Model

  • Content: Basic box.
  • Padding: Outside pushing the border away from the content; used to give the content breathing room; background colour is visible (order: padding-top, padding-right, padding-bottom, padding-left).
  • Border: Stroke around the edge of padding where the background colour stops.
  • Margin: Transparent layer outside the box pushing other boxes away; the space around an element that seperates other elements; can be a positive or negative value (order: margin-top, margin-right, margin-bottom, margin-left).
  • *Margin & Padding Shorthands:
  • 1 number: specifies a margin or padding on all 4 sides
  • 2 number: top/bottom & left/right
  • 3 number: top, left/right, bottom

Width & Height

  • %: Use percent for elements that should grow with the size of the browser, e.g columns, images
  • em: Use ems for elements that should grow with the size of the text, e.g. line-lengths
  • px: Use pixels for things that should never change size, e.g. logos, icons

Min & Max Heights

  • min-width: Force the box to be at least this wide, but can grow wider.
  • min-height: Force the box to be at least this tall, but can grow taller.
  • max-width: Force the box to never exceed this width, but can get narrower than it.
  • max-height: Force the box to never exceed this height, but can get shorter than it.

Display Property

  • Inline: Allows element on the same line; height and width properties have no effect (inline elements ex. a, strong, em).
  • Block: Forces the element to be on its own line regardless of its width (block elements ex. ul, p, header).
  • Inline-block: Element is formatted as an inline element but you can apply height and width values (inline-block elements ex. img, button).
  • None: The element is completely removed (hides elements).

Centering Elements

  • {text-align: center;} on line/inline-block element: If applied to the element itself, the text inside it will be centered. If applied to the parent element, the inline elements and the text inside them will be centered.
  • {text-align: center;} on a block element: Using text-align:center on a block level element will only affect the content inside it, but will not move the element box itself to the center of its parent.
  • {margin: 0 auto;} on a block element: If we have a block element, with a width, andwant it to center within its parent, we use auto margins. auto — tells the browser to automatically calculate a margin that's the same on the left and right.

Float & Clear

  • Text wrapping with float: Composed of a selector and one or many declarations surrounded by curly braces {}.
  • {float: left}: Move the element to the left, allowing text to wrap on the right side.
  • {float: right}: Move theelement to the right, allowing text to wrap on the left side.
  • {float: none}: Turn floating off.
  • Clearing floating content: Identifies the HTML element being targeted for styling.
  • {clear: left}: Go below all left floated elements.
  • {clear: right}: Go below all right floated elements.
  • {clear: both}: Go below all floated elements.
